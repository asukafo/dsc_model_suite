# HDMI 2.1 DSC Requirements (from HDMI Specification 2.1, Section 7.7 + 10.3.2)

## Overview

HDMI 2.1 uses **VESA DSC 1.2a** to enable resolutions beyond raw FRL bandwidth (e.g., 8K@120Hz, 10K@120Hz).
Compressed video is transported over **FRL (Fixed Rate Link)** at up to 48 Gbps (4 lanes × 12 Gbps).

## Mandatory DSC Features

| Requirement | Value / Constraint | Source |
|---|---|---|
| DSC Version | **VESA DSC 1.2a** (`dsc_version_minor = 2`) | §7.7.1 |
| Rate Control Mode | **CBR only** (HDMI FRL requires fixed bytes/line) | §6.5.1.4 |
| Block Prediction | **Must be enabled** (`BLOCK_PRED_ENABLE = 1`) | §7.7.2 |
| Full ICH Error Precision | **Must be enabled** (`FULL_ICH_ERR_PRECISION = 1`) | §7.7.1 |
| slice_height | **≥ 96 lines**, and `slice_height × vertical_slices = Vactive` | §7.7.5.2 |
| slice_width | **≤ 2720 pixels**, and `slice_width × slices = Hactive` | §7.7.5.1 |
| bpc | **8 bpc mandatory**; 10/12/16 bpc optional | §7.7.6 |
| bpp range | 8–12 bpp (typical for 4:4:4/4:2:2); max 16 bpp | §7.8.3.1 |
| Pixel formats | RGB 4:4:4, YCbCr 4:4:4, YCbCr 4:2:2, Native 4:2:0 | §7.7.1 |
| Slices | 1, 2, 4, 8, 12, or 16 columns | §7.7.1 |

## Key DSC vs HDMI Constraints

| DSC Parameter | HDMI Constraint |
|---|---|
| `pps_identifier` | Must be `0x00` |
| `dsc_version_minor` | Must be `2` (DSC 1.2a) |
| `convert_rgb` | `1` for RGB, `0` for YCbCr |
| `simple_422` | Used for simple 4:2:2 mode |
| `native_422` / `native_420` | Used for native chroma subsampling |
| `FULL_ICH_ERR_PRECISION` | Must be `1` |

## FRL Transport Layer

### HCactive Calculation

```
HCactivebytes = Slices × ceil(slice_width × bpp / 8)
HCactive = ceil(HCactivebytes / 3)  Tri-Bytes
```

- Zero-padding at end of Line of Chunks to fill 3-byte Tri-Byte boundary
- 6 cases for FRL packet loading based on `MOD(HCactivebytes, 6)` (§6.5.1.4)
- Gap Characters may be inserted during HCactive period

### Data Flow Metering

- Sources must meet Tri-Byte Data Flow Metering requirement (`TBDifference,Source ≤ 492 Tri-Bytes`)
- Sinks must tolerate `TBDifference,SinkTolerance ≥ 504 Tri-Bytes`
- `HCactiveTarget` and `HCblankTarget` computed via steps in Table 6-55/6-56/6-57

### FRL Link Rates

| Rate | Lanes | Total BW |
|---|---|---|
| 3 Gbps | 3 | 9 Gbps |
| 6 Gbps | 3 / 4 | 18 / 24 Gbps |
| 8 Gbps | 4 | 32 Gbps |
| 10 Gbps | 4 | 40 Gbps |
| 12 Gbps | 4 | 48 Gbps |

## Sink Capability Data Structure (SCDS)

From HF-VSDB (§10.3.2, Byte 8–10):

| Byte | Bits | Field | Description |
|---|---|---|---|
| Byte 8 | [0] | `DSC_10bpc` | 1 = supports 10 bpc DSC |
| Byte 8 | [1] | `DSC_12bpc` | 1 = supports 12 bpc DSC |
| Byte 8 | [2] | `DSC_16bpc` | Reserved (must be 0) |
| Byte 8 | [3] | `DSC_ALL_bpp` | 1 = all bpp settings in 1/16 steps |
| Byte 8 | [4] | `DSC_422_Option` | 1 = can adapt to 4:2:2 fallback |
| Byte 8 | [6] | `DSC_Native_420` | 1 = supports native 4:2:0 |
| Byte 8 | [7] | `DSC_1p2` | 1 = VESA DSC 1.2a compliant |
| Byte 9 | [7:4] | `DSC_Max_FRL_Rate` | Max FRL rate for DSC |
| Byte 9 | [3:0] | `DSC_MaxSlices` | Max slice units (1/2/4/8/12/16) |
| Byte 10 | [5:0] | `DSC_TotalChunkKBytes` | Line buffer in 1024-byte units |

## PPS Transmission

- PPS (128 bytes) transmitted via **CVTEM EMP** (Compressed Video Transport Extended Metadata)
- Sent once per video field/frame during vertical blanking
- `HCactive_bytes[15:0]` also transmitted in CVTEM for Sink to know exact bytes/line

## Primary Compressed Formats (Normative, Table 7-35)

For each CTA-861 video timing, specifies:
- FRL rate and lane count
- `bpp`, `HCactive`, `HCblank`

Sources must support at least the minimum FRL configuration listed.
If `BytesTarget > DSC_TotalChunkKBytes`, the configuration is not supported.

## 3D Video Support

- Frame Packing: each view compressed independently, same PPS
- Top-and-Bottom: same PPS, double vertical active
- Side-by-Side (Half): `slice_width = Hactive/2`, Slices = 2

## Relevant VESA DSC 1.2a Parameters for HDMI

From Annex D of VESA DSC spec (referenced by HDMI §7.7.5.3):

| Parameter | HDMI Guidance |
|---|---|
| `rc_model_size` | Per VESA DSC Annex D.3 |
| `initial_offset` | Rate buffer sizing per slice_width (see §7.7.4.1 figures) |
| `initial_xmit_delay` | Per C-Model |
| `chunk_size` | `ceil(slice_width × bpp / 8)` |
| `slice_bpg_offset` | Auto-generated by C-Model |
| `rc_buf_thresh[]` | Per VESA DSC RC parameter set |

## Notes for Performance Model

1. HDMI always uses **CBR** — no VBR mode needed for HDMI compliance
2. **`FULL_ICH_ERR_PRECISION = 1`** is mandatory — affects encode pipeline timing
3. **`BLOCK_PRED_ENABLE = 1`** is mandatory — BP search always runs
4. **slice_height ≥ 96** — minimum slice size is larger than typical DSC configs
5. Sink capabilities limit configuration at runtime — model should accept `DSC_MaxSlices` and `DSC_TotalChunkKBytes` as constraints
6. Data Flow Metering adds a system-level buffer constraint beyond the DSC rate buffer
